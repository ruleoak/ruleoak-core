# Security Policy

RuleOak Core v1.0 is a local-first governed AI runtime with a sandbox foundation. Security reports should not be filed as public issues.

## Reporting a vulnerability

Email the project maintainers with:

- impact;
- reproduction steps;
- affected version or commit;
- relevant logs or screenshots;
- suggested mitigation, if available.

Do not include real customer data, API keys, personal notes, or confidential examples. Use synthetic samples.

## Security boundary

RuleOak Core v1.0 includes filesystem, network, command, and tool policy guards. These are deny-by-default controls with automated tests and documentation.

This is not yet an externally security-reviewed sandbox, certified compliance product, or production security guarantee.

## Safe use guidance

Before using RuleOak with sensitive data or real systems, review:

- tool permissions;
- local model runner behavior;
- data retention paths;
- logs and generated reports;
- secrets handling;
- network access;
- organizational open-source policy.
