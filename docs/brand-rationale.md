# Brand Rationale

**RuleOak** combines two ideas:

```text
Rule = policy, boundaries, approval, governance, audit
Oak  = roots, strength, endurance, branching growth, long-term trust
```

RuleOak is designed for AI workflows where agents should be useful, but not reckless. The core runtime acts like the trunk of a tree: it gives every vertical workflow a shared foundation for policy, evidence, approval, and audit.

Brand line:

> One governed root. Many vertical workflows. Every important action accountable.

Why the domain and GitHub organization use the same name:

```text
ruleoak.com                 public brand and product website
github.com/ruleoak          developer home and open-core repositories
docs.ruleoak.com            future documentation home
app.ruleoak.com             future product home
```

This keeps the brand simple. Users, developers, and technical teams see one consistent identity across the website, documentation, open-source code, and future products.
