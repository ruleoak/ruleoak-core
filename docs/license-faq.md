# License FAQ

RuleOak Core v1.0 is licensed under **AGPL-3.0-or-later**.

This FAQ is not legal advice. Review the license and consult counsel for production or commercial use.

## Can I use RuleOak Core for learning and local experiments?

Yes. RuleOak Core is intended to be easy to run, inspect, and adapt.

## Can I build an open-source app with RuleOak Core?

Yes, if your app complies with AGPL requirements and uses a compatible license strategy.

## Can I use RuleOak Core inside a company?

Many companies require review before using AGPL software. Check your organization’s open-source policy before using RuleOak Core in internal systems or products.

## Can I embed RuleOak Core in a closed-source product?

AGPL obligations may apply if RuleOak Core is included, modified, distributed, or offered as a network service. A separate commercial license may be appropriate for closed-source embedding.

## Why AGPL?

RuleOak Core is infrastructure-like open core. AGPL helps keep improvements to the core available to users when modified versions are offered over a network, while still allowing a separate commercial licensing path for closed-source embedding.
