# Post-launch Feedback Loop

The first v2.2.0 launch goal is not maximum traffic. The goal is to discover whether developers understand RuleOak quickly.

## First feedback questions

Ask early users:

1. Could you explain RuleOak after the 10-minute quickstart?
2. Which demo made the value clearest?
3. Did the security boundary feel clear?
4. Did the demo GIF explain the flow quickly?
5. What adapter or connector would make RuleOak useful in your project?

## First 72 hours

- Watch install failures.
- Watch confusion around quickstart commands.
- Watch whether developers find the quickstart before long docs.
- Collect missing adapter requests.
- Collect missing evidence connector requests.
