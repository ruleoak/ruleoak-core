# GitHub Release Guidance for v2.2.0

## Release title

```text
RuleOak Core v2.2.0 — Governance for AI tool calls
```

## Release description

```text
RuleOak Core adds policy, approval, evidence records, audit trails, and local reports around AI tool calls.

It is designed for developers building AI agents, MCP-style tools, LangGraph/CrewAI workflows, local LLM workflows, and serious vertical applications where unchecked tool execution is not acceptable.
```

## Highlights

- 10-minute developer quickstarts
- Tool-call allow / approval-required / deny decisions
- Governance Protocol v1 conformance kit
- Versioned policy packs with scenario tests and signatures
- Approval UX v2 local workflow
- Audit Report Viewer v2
- Signed evidence and audit-chain integrity checks
- Reference verticals for coding agents, enterprise RAG, local assistants, and SRE monitoring change governance

## Do not say

- Compliance certified
- Enterprise ready
- Safe agents
- Replaces security controls
- Complete AI governance platform
- Claims full regulated-production hardening

## Say instead

- Audit-friendly
- Evidence-oriented
- Local-first
- Policy outside prompts
- Approval before risky actions
- Replayable governance records
- Path toward serious or regulated workflows
