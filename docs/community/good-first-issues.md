# Good First Issues

These are small, public-friendly tasks for new contributors. They are intentionally scoped to documentation, examples, and developer experience so contributors can help without understanding the full runtime internals.

## 1. Verify Windows quickstart commands

**Labels:** `docs, good first issue`


Test the README quickstart on Windows PowerShell and document any command differences.


Copy-ready issue: [good-first-issues/01-verify-windows-quickstart.md](good-first-issues/01-verify-windows-quickstart.md)


## 2. Add more policy example snippets

**Labels:** `examples, good first issue`


Add small policy examples for common actions such as publish_report, send_email, write_file, external_network_call, and delete_record.


Copy-ready issue: [good-first-issues/02-add-more-policy-examples.md](good-first-issues/02-add-more-policy-examples.md)


## 3. Improve HTML report accessibility

**Labels:** `reports, accessibility, good first issue`


Review generated HTML reports for heading order, alt text, keyboard navigation, and contrast. Propose small fixes.


Copy-ready issue: [good-first-issues/03-improve-html-report-accessibility.md](good-first-issues/03-improve-html-report-accessibility.md)


## 4. Expand the Python bridge walkthrough

**Labels:** `python, docs, good first issue`


Add a step-by-step walkthrough showing how a generic Python app emits RuleOak-compatible records and where those records are stored.


Copy-ready issue: [good-first-issues/04-expand-python-bridge-walkthrough.md](good-first-issues/04-expand-python-bridge-walkthrough.md)


## 5. Add a policy decision diagram

**Labels:** `docs, governance, good first issue`


Create a simple Mermaid or SVG diagram showing deny > approval_required > allow decision priority.


Copy-ready issue: [good-first-issues/05-add-policy-decision-diagram.md](good-first-issues/05-add-policy-decision-diagram.md)
