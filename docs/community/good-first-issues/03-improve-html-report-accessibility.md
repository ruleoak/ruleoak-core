# Improve HTML report accessibility

## Goal

Review generated HTML reports for heading order, alt text, keyboard navigation, and contrast. Propose small fixes.

## Suggested scope

- Keep the change small.
- Prefer documentation, examples, or tests.
- Do not change public APIs unless the maintainer agrees first.

## Acceptance criteria

- The change is easy to review.
- Existing tests still pass.
- README or docs remain clear for first-time users.

## Suggested labels

reports, accessibility, good first issue
