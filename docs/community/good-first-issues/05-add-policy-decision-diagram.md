# Add a policy decision diagram

## Goal

Create a simple Mermaid or SVG diagram showing deny > approval_required > allow decision priority.

## Suggested scope

- Keep the change small.
- Prefer documentation, examples, or tests.
- Do not change public APIs unless the maintainer agrees first.

## Acceptance criteria

- The change is easy to review.
- Existing tests still pass.
- README or docs remain clear for first-time users.

## Suggested labels

docs, governance, good first issue
