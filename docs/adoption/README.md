# Developer Adoption Kit

This kit is for the first 10 minutes with RuleOak.

Use it when a developer asks: "Where do I put RuleOak in my app?"

The answer is simple:

```text
existing tool call
→ RuleOak evaluates policy
→ allowed / approval_required / blocked
→ evidence and audit records are written
→ the host app executes only if allowed or approved
```

Start here:

```bash
npm install
npm run quickstart:all
npm run adoption:real-frameworks
```

Public-release note: the latest public RuleOak Core release is **v2.2.0**. Developer adoption assets are included in this release.
