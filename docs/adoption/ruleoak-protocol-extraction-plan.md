# ruleoak-protocol extraction plan

Create `ruleoak-protocol` after the v2.2.0 story stabilizes. It should contain only protocol-level material:

- Evidence JSONL v1 schema
- `.ruleoak.yml` v1 schema
- skill manifest v1 schema
- adapter action envelope types
- conformance fixtures
- badge definitions
- minimal validators

This lowers adoption friction while preserving the RuleOak Core runtime moat.
