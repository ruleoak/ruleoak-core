# OpenClaw-style security disclosure process

Use private disclosure for unpatched vulnerabilities, exploit paths, or instructions that could help misuse a real agent project.

## Public content may include

- generic integration pattern
- mock dangerous action demos
- safe permission/evidence architecture

## Public content must not include

- exploit chains against a live project
- real credentials, filesystem paths, or private database examples
- instructions to bypass current protections

## Private advisory checklist

1. affected version/commit if known
2. risk category
3. safe reproduction using mocks when possible
4. impact
5. suggested mitigation
6. offer optional RuleOak integration pattern
