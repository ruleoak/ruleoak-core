# Personal Local-First Assistant Governance

This RuleOak Core v2.2.0 reference vertical shows RuleOak governing a personal local-first assistant.

Run:

```bash
npm run personal:local-assistant-governance
```

The workflow demonstrates:

- local note and calendar summary reads are allowed;
- external email send requires human approval;
- private-file upload is blocked;
- local-first evidence, privacy scan evidence, approval records, audit events, and reports are generated.

This vertical is designed for indie developers and privacy-first app builders who want personal AI assistants to draft locally while gating external actions.

Boundary statement: RuleOak governs assistant actions and records. It does not replace operating-system permissions, provider permissions, or user-consent flows.
